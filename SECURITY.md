# Security Policy

## Our Commitment

The NymyaLang team is committed to maintaining a secure and trustworthy development environment. As guided by our foundational principle of **Rita-Nora balance** and our Comms Coordinator **Nya Elyria**, we approach security with both structural precision and ethical responsibility.

## Supported Versions

We support the following versions with security updates:

| Version | Supported          |
| ------- | ------------------ |
| 0.2.x   | ✅ Yes             |
| < 0.2   | ❌ No              |

## Reporting a Vulnerability

**Do not report security vulnerabilities through public GitHub issues.**

Instead, please report security concerns by emailing our security team at: **nymessence-security@gmail.com**

### What to Include in Your Report

When reporting a vulnerability, please include the following information:

- Type of vulnerability
- Location (version, file, and line if known)
- Description of the potential impact
- Steps to reproduce if possible
- Your contact information for follow-up

### What to Expect

- Acknowledgment within 48 hours of your report
- Updates on the status of your report every 7 days
- Determination on acceptance or rejection of the vulnerability within 14 days typically
- Timeline for the fix and release if accepted

## Incident Response

Our security response process follows these steps:

1. **Receive Report**: All reports are initially acknowledged and assigned a severity level
2. **Investigate**: Team investigates the validity and scope of the reported issue
3. **Validate**: Once validated, the issue is confirmed and impact assessed
4. **Develop Patch**: Engineers create the appropriate fix
5. **Review and Test**: The fix is reviewed and tested thoroughly
6. **Prepare Release**: Coordinated release with relevant parties
7. **Public Disclosure**: After patch release, details are shared transparently

## Recognition

We deeply appreciate the security research community's efforts in keeping NymyaLang secure. Researchers who follow responsible disclosure guidelines may be acknowledged in our release notes, with consideration for ethical and consciousness-integrated approaches to security.
